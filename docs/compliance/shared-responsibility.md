# Shared responsibility

The repository can implement technical controls, but certification and regulatory compliance belong to an operating organization.

Microsoft secures the underlying Azure cloud platform according to its service responsibilities. The deploying organization remains responsible for identity governance, configuration, application code, data classification, legal basis, access review, monitoring, incident response, change control, vendor management, recovery testing, and evidence.

A successful deployment does not independently establish ISO certification, SOC 2 assurance, SEC compliance, FINRA compliance, or suitability for real client data.
