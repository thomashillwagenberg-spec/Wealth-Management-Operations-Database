# Screenshot capture guide

Do not add a screenshot until the project has actually run and the displayed result has been verified. Never fabricate a result, execution plan, validation grid, backup message, or Azure deployment.

## Before capturing

- Use a clean SSMS window.
- Increase the query and result font enough for mobile viewing.
- Close unrelated tabs.
- Hide server names, Windows usernames, local paths, IP addresses, subscription details, and credentials where they create privacy or security risk.
- Keep the database name and relevant object names visible.
- Show enough SQL to establish what produced the result.
- Do not crop away an error message that changes the meaning.
- Prefer PNG.
- Use descriptive filenames.
- Add short captions in the README only after the files exist.

## Required screenshot checklist

### 1. Object Explorer architecture

**Filename:** `01-object-explorer.png`

Show:

- `WealthManagementOperations`
- Tables expanded enough to reveal the six schemas
- Views and Programmability visible if space permits

Do not show:

- Unrelated databases
- Server details you do not want public
- Connection credentials

**Insert in README after:** Database architecture.

---

### 2. Relationship design

**Filename:** `02-relationship-design.png`

Option A:

- Use SSMS Database Diagrams if installed and enabled.
- Arrange tables by schema.
- Make primary and foreign-key relationships readable.

Option B:

- Render the Mermaid diagram in `docs/database-design.md`.
- Capture the rendered relationship map.

Do not claim the visual is generated by SSMS when it is Mermaid.

**Insert in README after:** Link to database design.

---

### 3. Portfolio-value query

**Filename:** `03-portfolio-value-report.png`

Run the first query in `sql/12_analysis_queries.sql`.

Show:

- Query title/comment
- Client code and fictional client name
- Account count
- Total portfolio value
- Unrealized gain/loss
- Sort order

Verify that all values are synthetic.

**Insert in README after:** Example business questions.

---

### 4. Risk or compliance report

**Filename:** `04-risk-compliance-report.png`

Recommended:

```sql
SELECT *
FROM reporting.vw_RiskAlignment
WHERE AlignmentStatus <> 'ALIGNED'
ORDER BY DeviationPctPoints DESC;
```

Alternative:

```sql
SELECT *
FROM reporting.vw_ComplianceDashboard
WHERE RequiresReview = 1;
```

Show the calculated status and enough columns to explain why review is required.

**Insert in README after:** Security and privacy or business questions.

---

### 5. Stored procedure execution

**Filename:** `05-stored-procedure.png`

Run:

```sql
EXEC reporting.usp_ClientPortfolioReport
    @ClientID = 1,
    @AsOfDate = '2026-07-10';
```

Show the procedure call and output grid.

**Insert in README after:** Skills demonstrated.

---

### 6. Execution plan and index comparison

**Filename:** `06-execution-plan.png`

1. Enable **Include Actual Execution Plan**.
2. Run the exercise in `sql/10_create_indexes.sql`.
3. Show the relevant operator tooltip or properties.
4. Capture logical reads in a separate image if necessary.

Do not write “X% faster” unless you have repeatable before-and-after measurements.

**Insert in README after:** Skills demonstrated or future improvements.

---

### 7. Validation success

**Filename:** `07-validation-results.png`

Run `sql/13_validation_tests.sql`.

Show:

- Test names
- Expected outcomes
- Actual outcomes
- Status column
- The completion message

Do not publish this screenshot if any test shows `FAIL`. Explain any `MANUAL` result.

**Insert in README after:** Testing instructions.

---

### 8. GitHub repository overview

**Filename:** `08-github-repository.png`

Show:

- Repository name
- Description
- Folder structure
- README opening
- Public visibility only after content review

Do not show private email, personal access tokens, secret-scanning alerts with secret values, or browser tabs unrelated to the project.

## Caption format

Use one sentence per image:

- “The repository is organized as an ordered, reproducible SQL Server build.”
- “Six schemas separate client, market, trading, compliance, audit, and reporting responsibilities.”
- “Portfolio values are calculated from current quantities and the latest eligible synthetic prices.”
- “Risk alignment compares equity-like allocation with the client’s synthetic profile range.”
- “The validation suite independently checks counts, constraints, holdings, calculations, and permissions.”

## Completion check

Every screenshot should answer:

1. What am I showing?
2. What business or technical skill does it prove?
3. Is the result real?
4. Is the data synthetic?
5. Have I hidden unnecessary sensitive details?
